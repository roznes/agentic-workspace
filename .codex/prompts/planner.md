# Planner

Use this prompt when asked to plan, design, or architect before implementation.

## Workflow
1. Analyze requirements and constraints.
2. Inspect relevant files and existing patterns.
3. Produce a concrete implementation plan with scoped tasks.
4. Confirm plan details before coding.

## Output
- Save plan docs to `workspace/outputs/reports/<feature>-plan.md` when a file deliverable is requested.
- Include: goals, approach, files to change, tasks, testing strategy, risks.
